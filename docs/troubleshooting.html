Troubleshooting & FAQ

Common issues and how to solve them. If you don't find your answer here, check the GitHub issues page.

Common Issues

❌ Addon isn't showing in Blender

Problem: You installed the addon but don't see the Easy Rigify tab.

Solution:

  1. Go to Edit → Preferences → Get Extensions
  2. Search for "Easy Rigify"
  3. Make sure the checkbox is ticked (enabled)
  4. If it's grayed out, click it to enable
  5. Close and reopen Preferences
  6. The tab should now appear in the 3D Viewport sidebar (press N)

❌ "Rigify addon not found"

Problem: Easy Rigify says Rigify is disabled.

Solution:

  1. Go to Edit → Preferences → Add-ons (not Get Extensions)
  2. Search for "Rigify"
  3. Check the box next to it to enable
  4. Close Preferences
  5. Try again
ℹ️ Note: Rigify is built into Blender but needs to be enabled. Easy Rigify won't work without it.

❌ Markers won't place on my mesh

Problem: Clicking "Auto Detect" doesn't place markers.

Solution:

  • Make sure you've selected your character mesh
  • Check that "Body Mesh" is set in the Markers panel
  • Apply all transforms: Object → Apply → All Transforms
  • Make sure your mesh isn't too small or too large (try scaling to ~1-2 Blender units tall)
  • Check that your mesh is a single connected object (not multiple objects)

❌ Alignment is completely off

Problem: Bones don't match markers at all.

Solution:

  1. Delete the metarig (it's called "metarig")
  2. Apply all transforms on your mesh
  3. Re-detect and place markers
  4. Run "Check All Markers" to validate
  5. Run "Align Rig to Markers" again

Why this happens: Unapplied scale/rotation is the #1 cause of bad alignment.

❌ AI detection doesn't work (Full Edition)

Problem: "EasyDetect" buttons don't work or show errors.

Solution:

  • The AI runtime might not have loaded. Check the system console: Help → Toggle System Console
  • Look for error messages mentioning "onnxruntime" or "AI"
  • Try switching to geometric detection as a workaround
  • AI may not work on exotic platforms (very old GPU, ARM processor, etc.)
💡 Fallback: If AI doesn't work, geometric detection still works perfectly. You just won't get the AI advantage.

❌ Mesh deformation looks bad

Problem: The rig moves but the mesh collapses, pinches, or twists weirdly.

Solution:

  1. Make sure you applied "Tighten Weights" in the Skin tab
  2. Try a different tightening preset (Gentle, Default, Aggressive)
  3. Use "Analyze Joint Bleed" and "Repair Joint Bleed" to fix problem areas
  4. Manually paint weights on problem joints
  5. Use "Smooth Weights" to blend transitions

❌ Fingers look wrong or twisted

Problem: Finger markers are misplaced or rotated incorrectly.

Solution:

  • Use "Straighten Fingers" to auto-correct wander
  • Manually re-detect just that finger using the per-finger picker
  • Make sure your hand is in a clear, neutral pose (not curled or twisted)
  • For the full edition, try "EasyDetect Fingers" for better accuracy

❌ Shoulder or elbow keeps pinching

Problem: Deformation looks creased at arm joints.

Solution:

  1. Use "Tighten Weights" → select "Shoulders" or "Forearms"
  2. Try "Aggressive" preset instead of Default
  3. Manually paint weights to smooth the transition between bones
  4. Use the brush to blend weights across the joint area

❌ "Check All Markers" shows errors

Problem: Missing, overlapping, or outside-mesh markers.

Solution:

  • Missing: Place the marker in the position shown by the on-screen hint
  • Overlapping: Markers are too close together; move them apart
  • Outside mesh: Marker is floating in empty space; grab (G) and move it onto the mesh

❌ Blender crashes when I click a button

Problem: Blender freezes or crashes during detection or rigging.

Solution:

  • Save your work first!
  • Make sure your mesh doesn't have millions of vertices (decimation might help)
  • Try closing and reopening Blender
  • Update to the latest Blender version (4.2 or newer)
  • Check the GitHub issues page for your specific Blender version

Best Practices to Avoid Issues

Pre-Rigging Checklist

  • ✅ Character in T-pose or A-pose
  • ✅ Applied all transforms (Object → Apply → All Transforms)
  • ✅ Rigify addon is enabled
  • ✅ Mesh is selected
  • ✅ Mesh is a single connected object (not multiple)
  • ✅ Character is roughly 1-3 Blender units tall

During Marker Placement

  • ✅ Use auto-detection first (faster)
  • ✅ Validate with "Check All Markers" before proceeding
  • ✅ Markers should look visually correct in the viewport
  • ✅ Use Live Symmetry to maintain bilateral accuracy

After Rigging

  • ✅ Test the rig in Pose Mode immediately
  • ✅ Apply "Tighten Weights" before manual painting
  • ✅ Test deformation while painting (don't wait until the end)
  • ✅ Save frequently

Performance Tips

  • 💨 Detection is faster on simpler meshes (high poly = longer detection time)
  • 💨 Use geometric detection first; it's faster than AI
  • 💨 Close unnecessary viewports/windows to free up GPU memory
  • 💨 Disable preview rendering while adjusting markers

Frequently Asked Questions

Q: Can I use Easy Rigify with Rigify's other features?

A: Yes! Easy Rigify creates a standard Rigify rig. You can manually tweak the control shapes, add custom controls, or use any other Rigify features after generation.

Q: Do both editions work together?

A: Yes. A .blend file saved in the Full Edition opens in Lite and uses geometric detection. You can switch between them (one at a time) for the same project.

Q: Can I use this for non-humanoid characters?

A: Not officially. Easy Rigify is built for humanoid characters (bipeds with arms, legs, hands, and face). Quadrupeds or alien anatomy would need manual marker placement.

Q: What's the maximum mesh size?

A: No hard limit, but detection gets slower with very high poly counts (millions of vertices). Consider decimating for faster detection.

Q: Can I animate right after rigging?

A: Yes, but we recommend testing the rig's deformation first (pose a few test positions). If it looks good, you're ready to animate.

Q: Does it work with character assets from 3D marketplaces?

A: Usually yes! As long as the character is humanoid and the mesh is clean (no floating bits, correct normals), Easy Rigify should work.

Q: Can I export the rig to game engines?

A: Yes. Export as FBX or glTF from Blender. The Rigify rig is a standard armature and works with any engine that supports skeletal animation (Unity, Unreal, etc.).

Q: Is internet needed?

A: No. Everything runs on your machine. The full edition's AI runtime is bundled and runs locally.

Q: How do I report a bug?

A: Go to the GitHub repository and open an issue. Include:

  • Your Blender version
  • Steps to reproduce
  • Screenshot or error message
